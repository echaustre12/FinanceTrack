# Historias de usuario definidas

### HU-01
Como usuario de la plataforma quiero registrar mis ingresos para tener un presupuesto claro.
### HU-02
Como usuario de la plataforma quiero registrar los gastos realizados con distintos medios de pago.
### HU-03
Como usuario de la plataforma quiero dividir mis gastos en categorias para mejor organización.
### HU-04
Como usuario de la plataforma quiero recibir notifiaciones para realizar pagos oportunamente.
### HU-05
Como usuario de la plataforma quiero organizar un presupuesto por categorias para saber cuando gasto más de lo debido.
### HU-06
Como usuario de la plataforma quiero tener registro de todos mis datos financieros mensuales para llevar un seguimiento.
### HU-07
Como usuario quiero que el sistema cree automáticamente un nuevo periodo financiero cada mes para comenzar un nuevo seguimiento sin perder mi historial anterior.
### HU-08
Como usuario de la plataforma quiero tener un calendario con todos los pagos de mas de 100.000 COP que he realizado hasta el momento para tener una representacion grafica de mis gastos importantes.
### HU-09
Como usuario de la plataforma quiero que mis datos esten seguros para no tener riesgos sobre filtracion de mis gastos, o rutinas economicas.
### HU-10
Como usuario de la plataforma quiero ver graficas como balance general, grafica de gastos, grafica de ingresos, grafica de ahorros e indicadores en lo que gaste mas de lo esperado para saber en que debo mejorar.
### HU-11
Como usuario de la plataforma quiero poder registrar pagos desde WhatsApp o similares para no tener que abrir la plataforma desde un computador.
### HU-12
Como usuario de la plataforma quiero saber cuanto dinero me queda por categoria con consultas desde el mismo bot para no tener que abrir la plataforma desde el computador.
### HU-13
Como usuario de la plataforma quiero tener graficas comparativas de mis gastos anteriores para hacer un analisis de los gastos.
### HU-14
Como usuario de la plataforma quiero registrar cuando recibo mis pagos, los cuales no se recibiran de manera regular, para saber cuanto dinero del presupuesto total tengo realmente.
### HU-15
Como usuario de la plataforma quiero que el sistema maneje un modelo predictivo que muestre en que días del mes recibo normalmente dinero para acomodar mis gastos.
### HU-16
Como usuario quiero registrar pagos recurrentes para que el sistema los genere automáticamente cada mes.
### HU-17
Como usuario quiero definir metas de ahorro para monitorear mi progreso financiero.

# Requisitos Funcionales y No Funcionales

### RQ-01
El sistema debe integrar microservicios.
### RQ-02
El sistema debe proteger los datos de los usuarios con Spring Security.
### RQ-03
El sistema debe utilizar Spring boot y JPA para manejar las entidades. 
### RQ-04
El sistema debe manejar testing automatizado con GitHub Actions.
### RQ-05
El sistema debe contener pipelines CI/CD.
### RQ-06
El sistema debe desplegarse usando Docker y Kubernetes.
